# Required Documents

## Purpose

This document defines the complete checklist of documents required throughout the PMKSY project lifecycle for a chocolate manufacturing unit. It serves as the single authoritative reference for document requirements, organized by implementation stage.

**OFFICIAL** — PMKSY applications require comprehensive documentation to establish eligibility, technical feasibility, financial viability, and compliance. Missing or incomplete documentation is a leading cause of application rejection.

> **SCOPE BOUNDARY**: This document addresses **document requirements only** — what documents are needed, when, and in what form. It does **not** address:
> - Eligibility criteria (see [PMKS-002: Project Eligibility](PMKSY_MASTER/02_PROJECT_ELIGIBILITY.md))
> - DPR preparation methodology (see [PMKS-008: Application Workflow](PMKSY_MASTER/08_APPLICATION_WORKFLOW.md))
> - Document legal standards (consult your legal advisor)

---

## Document Lifecycle Stages

**PROJECT ASSUMPTION** — Documents are required across six primary stages:

1. **Pre-application** — Documentation before formal application submission
2. **Application** — Documents submitted with PMKSY application
3. **Bank appraisal** — Documents for bank term loan processing
4. **Sanction** — Documents for PMKSY sanction and agreement execution
5. **Grant release** — Documents for subsidy disbursement
6. **Post-project monitoring** — Documents during and after project implementation

## Document Preparation Guidelines

### DPR Preparation

**ENGINEERING RECOMMENDATION** — The DPR should include or reference all application-stage documents. Prepare DPR using MoFPI template if available.

**PROJECT ASSUMPTION** — DPR should be prepared by a registered technical consultant with:
- Food processing industry experience
- PMKSY scheme familiarity
- Chocolate/confectionery project experience (preferred)

### Document Organization

**ENGINEERING RECOMMENDATION** — Maintain a project document register with:

| Field | Description |
|-------|-------------|
| Document name | Full document title |
| Document ID | Unique internal reference |
| Issuing authority | Source organization |
| Issue date | Date of issuance |
| Validity period | Expiry or renewal date |
| Physical location | Storage location (original/copy) |
| Stage required | Which PMKSY stage needs it |
| Renewal action | Reminder for expiry |

**PROJECT ASSUMPTION** — Use a document management system or cloud storage with access controls for team collaboration.

### Document Authentication

**OFFICIAL** — Documents may require:
- Self-attestation by applicant
- Notarization (for certain documents)
- Apostille / legalization (for foreign documents)
- Chartered accountant certification (for financial documents)
- Bank attestation (for bank statements)

**PROJECT ASSUMPTION** — For a ₹2 crore chocolate manufacturing project in Andhra Pradesh, notarization and CA certification are typically sufficient. Aportille not required for Indian documents.

## Document Submission Best Practices

### Submission Format

**PROJECT ASSUMPTION** — Submit documents as:
- Indexed and bound hard copy (2-3 sets)
- PDF soft copy on USB/drive or email
- Scanned copies for preliminary reference

**ENGINEERING RECOMMENDATION** — Number all pages consecutively. Use table of contents for large submissions.

### Common Submission Mistakes

### Mistake 1: Incomplete Index
- **Error**: Submitting documents without clear index
- **Prevention**: Create document checklist (this document) and tick off each item

### Mistake 2: Expired Documents
- **Error**: Submitting certificates past validity
- **Prevention**: Check validity dates; renew before submission

### Mistake 3: Self-Attestation Issues
- **Error**: Forgetting self-attestation or using wrong format
- **Prevention**: Self-attest all copies with signature and date

### Mistake 4: Missing Supporting Documents
- **Error**: Submitting main document without supporting annexures
- **Prevention**: Submit complete application package with all annexures

### Mistake 5: Inconsistent Information
- **Error**: Discrepancies between documents (e.g., different address formats)
- **Prevention**: Verify consistency across all documents before submission

### Mistake 6: Poor Quality Copies
- **Error**: Illegible, smudged, or incomplete copies
- **Prevention**: Use high-quality scanning; verify legibility

### Mistake 7: Missing Signatures/Stamps
- **Error**: Documents missing required signatures or stamps
- **Prevention**: Check each document for all required endorsements

## Relationship with DPR Preparation

**PROJECT ASSUMPTION** — DPR preparation and document collection should proceed in parallel:

| DPR Section | Supporting Documents |
|-------------|---------------------|
| Executive summary | Project concept, promoter profile |
| Project description | Land documents, NOCs |
| Technical feasibility | Technology agreements, equipment quotes |
| Market analysis | Market study, customer letters |
| Financial projections | Financial statements, cost estimates |
| Implementation schedule | Land documents, approvals |
| Compliance | Statutory clearances |

**ENGINEERING RECOMMENDATION** — Begin document collection at project conception. Many documents (land title, NOCs, environmental clearance) require 2-6 weeks to obtain.

## Relationship with Bank Appraisal

**BANK PRACTICE** — Banks require the same core documents as PMKSY with additional bank-specific requirements:

| PMKSY Document | Bank Equivalent | Difference |
|----------------|-----------------|------------|
| DPR | Bank-appointed DPR or project report | Bank may appoint its own consultant |
| Appraisal letter | Bank term loan sanction | PMKSY subsidy does not reduce bank appraisal quantum |
| Cost estimates | Bank-appointed valuer report | Independent verification |
| Legal documents | Legal opinion on title/entity | Bank-appointed advocate |

**REQUIRES RE-VERIFICATION** — Confirm whether all banks accept MoFPI in-principle approval or require independent appraisal.

## Document Retention Requirements

### Retention Period

**PROJECT ASSUMPTION** — Retain documents for:

| Document Type | Retention Period | Basis |
|---------------|------------------|-------|
| Grant-related documents | 7-10 years after project completion | Audit requirements |
| Statutory licenses | Duration of license + 3 years | Regulatory requirements |
| Financial records | 8 years after assessment year | Income Tax Act |
| Land documents | Permanent | Property rights |
| PMKSY agreement | Duration + 5 years | Contract obligations |

**REQUIRES RE-VERIFICATION** — Confirm exact retention requirements under current PMKSY agreement and applicable laws.

### Storage

**ENGINEERING RECOMMENDATION** — Maintain:
- Original documents in fireproof safe / bank locker
- Certified copies at project site
- Digital scans in cloud storage with backup
- Document register with expiry tracking
